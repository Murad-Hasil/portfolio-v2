## Entity: Contact

Stores enquiries submitted via the portfolio contact form.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| `id` | INTEGER | PK, auto-increment | |
| `name` | VARCHAR(100) | NOT NULL | Visitor's full name |
| `email` | VARCHAR(255) | NOT NULL, valid email format | |
| `subject` | VARCHAR(200) | NOT NULL | |
| `service` | VARCHAR(50) | NOT NULL | One of: AI Chatbot, Automation, Web App, RAG System, Other |
| `message` | TEXT | NOT NULL, min 20 chars | |
| `reference` | VARCHAR(20) | UNIQUE, NOT NULL | Format: YYYYMMDD-XXXX |
| `ip_address` | VARCHAR(45) | NULLABLE | For rate limiting; NOT stored in production |
| `created_at` | TIMESTAMP | NOT NULL, default now() | UTC |

**Validation rules**:
- `email`: must pass RFC 5322 format validation
- `message`: minimum 20 characters
- `service`: must be one of the 5 defined values
- Rate limit: max 3 submissions per IP per hour (enforced at API layer, not DB)

**State transitions**: None — contact submissions are immutable once created.

## Entity: ChatSession

Represents a single visitor's conversation session with the chatbot.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| `id` | INTEGER | PK, auto-increment | |
| `session_id` | UUID | UNIQUE, NOT NULL | Generated by frontend, stored in sessionStorage |
| `created_at` | TIMESTAMP | NOT NULL, default now() | UTC |
| `messages_count` | INTEGER | NOT NULL, default 0 | Incremented on each message |

## Entity: ChatMessage

Individual messages within a chat session (both user and assistant).

| Field | Type | Constraints | Notes |
|---|---|---|---|
| `id` | INTEGER | PK, auto-increment | |
| `session_id` | UUID | FK → ChatSession.session_id | |
| `role` | VARCHAR(10) | NOT NULL | One of: user, assistant |
| `content` | TEXT | NOT NULL | Message text |
| `tokens_used` | INTEGER | NULLABLE | Populated for assistant messages only |
| `created_at` | TIMESTAMP | NOT NULL, default now() | UTC |

**Retrieval**: Last 10 messages per session fetched for chat history context window.

## Entity: PageView

Anonymous page visit tracking for basic analytics.

| Field | Type | Constraints | Notes |
|---|---|---|---|
| `id` | INTEGER | PK, auto-increment | |
| `page` | VARCHAR(100) | NOT NULL | e.g., `/`, `/projects/crm-digital-fte` |
| `referrer` | VARCHAR(500) | NULLABLE | HTTP Referer header |
| `created_at` | TIMESTAMP | NOT NULL, default now() | UTC |
| `country` | VARCHAR(2) | NULLABLE | ISO 3166-1 alpha-2, derived from IP (not stored) |

**Privacy**: IP address is NEVER stored. Country may be derived from IP at request
time and stored, but the raw IP is discarded immediately.

## Entity: Project (file-based, not DB)

Projects are NOT stored in PostgreSQL. They are read from
`context/projects-manifest.json` at runtime.

| Field | Type | Notes |
|---|---|---|
| `id` | string | URL-safe slug |
| `title` | string | Display title |
| `slug` | string | URL path segment |
| `status` | string | `completed` \| `in-progress` |
| `featured` | boolean | Show in featured section |
| `category` | string | AI Agents \| Full-Stack + AI \| AI Automation \| AI / Frontend |
| `description` | string | One-paragraph summary |
| `problem` | string | Business problem solved |
| `solution` | string | Technical approach summary |
| `tech` | string[] | Technology names |
| `live_url` | string \| null | Production URL |
| `github_url` | string \| null | GitHub repo URL |
| `image` | string | Path under `/public/projects/` |
| `metrics` | string[] | Measurable outcomes |
| `highlights` | string[] | Technical highlights |

**Source**: `context/projects-manifest.json` — read by both FastAPI `/projects`
endpoint and `scripts/seed-rag.py`.

## Entity: KnowledgeChunk (vector DB — Qdrant)

Text fragments from the RAG knowledge base stored as vector embeddings.

| Field | Notes |
|---|---|
| `id` | UUID point ID in Qdrant |
| `vector` | 384-dim float[] (fastembed) OR 1536-dim float[] (OpenAI) |
| `payload.text` | Raw chunk text |
| `payload.source` | Source filename (e.g., `projects.md`) |
| `payload.chunk_index` | Index within source file |
| `payload.embedding_provider` | `fastembed` \| `openai` |

**Collection name**: `portfolio-knowledge`
**Distance metric**: Cosine similarity
**Note**: Collection must be deleted and recreated when switching embedding
providers (dimension mismatch). The `rebuild_rag()` MCP tool handles this.

## Database Migration Strategy

- **Tool**: Alembic (auto-generate from SQLModel metadata)
- **Initial migration**: Creates all 4 tables (contacts, chat_sessions,
  chat_messages, page_views)
- **Rollback**: Each migration has a `downgrade()` function
- **Neon**: Uses connection pooling URL for application queries,
  direct URL for Alembic migrations (Neon recommends this split)
